## GitHub とは？

GitHub（ギットハブ）は、**ファイルをインターネット上に保存できるサービス**です。

- パソコンが壊れても、ファイルが消えにくくなる
- 別のパソコンからも同じファイルにアクセスできる
- ファイルの変更履歴を残せる

**重要なポイント：**  
GitHub にファイルを送っても、**パソコンの中のファイルは消えません**。コピーがインターネット上に保存されるだけです。

## 準備するもの

### 1. GitHub アカウント
- インターネットで「GitHub」と検索して、無料アカウントを作成してください

### 2. Personal Access Token（パーソナルアクセストークン）
GitHub にファイルを送るための「パスワードのようなもの」です。

#### 作り方：
1. GitHub にログイン
2. 右上のアイコンをクリック → **Settings**（設定）
3. 左側のメニューの一番下 → **Developer settings**
4. **Personal access tokens** → **Tokens (classic)**
5. **Generate new token (classic)** をクリック
6. **Note**（メモ）に「PC ファイルバックアップ」など、わかりやすい名前を入力
7. **Expiration**（有効期限）を設定（長く使いたい場合は長めに設定）
8. **Select scopes**（権限）で **`repo`** にチェックを入れる
9. 一番下の **Generate token** をクリック
10. 表示されたトークン（`ghp_` で始まる文字列）をコピーする
   - ⚠️ **この画面を閉じると二度と表示されないので、必ずコピーしてください！**

### 3. トークンをパソコンに登録する
コマンドプロンプトまたは PowerShell を開いて、以下のコマンドを実行：

```
setx GITHUB_TOKEN コピーしたトークン
```

例：
```
setx GITHUB_TOKEN ghp_xxxxxxxxxxxxxxxxxxxx
```

⚠️ **重要：** このコマンドを実行した後、**新しいコマンドプロンプト（または PowerShell）を開き直してください**。古いウィンドウでは反映されません。

## 最初の設定（1回だけ）

GitHub にファイルを送る前に、最初に1回だけ設定を行います。

### 手順：

1. コマンドプロンプトを開く
2. 以下のコマンドを実行：

```
cd C:\doc
setup_git.bat "送りたいフォルダのパス"
```

**例：**
```
setup_git.bat "C:\doc\taka\mirai_taka"
```

これで、そのフォルダが GitHub に送る準備が整います。

## ファイルを GitHub に送る方法

### 基本的な流れ：

1. コマンドプロンプトを開く
2. 以下のコマンドを実行：

```
cd C:\doc
push_git.bat "送りたいフォルダのパス"
```

**例：**
```
push_git.bat "C:\doc\taka\mirai_taka"
```

これで、フォルダ内のファイルが自動的に GitHub に送られます。

### 何が起こるか：

1. フォルダ内の変更されたファイルを自動的に検出
2. 日付と時刻が入ったメッセージで自動的に保存（コミット）
3. GitHub に送信（プッシュ）

**注意：** 変更がない場合は「コミット対象の変更がありません」と表示されますが、これは正常です。

## よくある質問

### Q1. フォルダを変更したい場合は？

`push_git.bat` のコマンドで、送りたいフォルダのパスを変えるだけです。

```
push_git.bat "C:\doc\別のフォルダ"
```

### Q2. 初めて送るフォルダの場合は？

最初に `setup_git.bat` を実行してから、`push_git.bat` を実行してください。

```
setup_git.bat "C:\doc\新しいフォルダ"
push_git.bat "C:\doc\新しいフォルダ"
```

### Q3. GitHub に送ったファイルは、パソコンから消える？

**いいえ、消えません。**  
GitHub に送っても、パソコンの中のファイルはそのまま残ります。コピーがインターネット上に保存されるだけです。

### Q4. 間違えて送ってしまったファイルを削除したい場合は？

GitHub のウェブサイトで、該当するコミット（保存記録）を「リバート」することで、削除できます。

1. GitHub のリポジトリページを開く
2. ファイル一覧の上にある「○ Commits」をクリック
3. 削除したいコミットをクリック
4. 右上の **Revert** ボタンをクリック
5. 表示された画面で **Create pull request** → **Merge pull request** をクリック

### Q5. エラーが出た場合は？

よくあるエラーと対処法：

- **「環境変数 GITHUB_TOKEN が設定されていません」**  
  → トークンの設定を確認してください（新しいコマンドプロンプトを開き直しましたか？）

- **「403 エラー」**  
  → トークンの権限（`repo` にチェックが入っているか）を確認してください

- **「non-fast-forward」**  
  → リモート（GitHub）に新しい変更があるため、先に取り込む必要があります：
  ```
  cd フォルダのパス
  git pull --rebase origin main
  ```
  その後、再度 `push_git.bat` を実行してください

## まとめ

1. **GitHub アカウントを作る**
2. **Personal Access Token を取得して、パソコンに登録する**
3. **初めて送るフォルダは `setup_git.bat` で設定**
4. **`push_git.bat` でファイルを送る**

これだけ覚えれば、基本的な使い方は完璧です！

## 補足：用語集

- **リポジトリ（Repository）**：GitHub 上でファイルを保存する場所。フォルダのようなもの
- **コミット（Commit）**：ファイルの変更を保存すること。「保存ボタン」のようなもの
- **プッシュ（Push）**：パソコンから GitHub にファイルを送ること
- **プル（Pull）**：GitHub からパソコンにファイルを取ってくること
- **ブランチ（Branch）**：ファイルのバージョンのようなもの。通常は `main` を使います
